# TIL - 코드 컨벤션 재정리와 AWS RDS 기반 DB 서버 확인

## 1. 진행 내용

코드 작성 시 맞춰야 할 기준을 다시 한 번 팀 내에서 정리했다.  
단순히 스타일만 맞추는 것이 아니라, 이후 협업 과정에서 코드 해석 방식과 작성 흐름이 흔들리지 않도록 기준을 통일하는 데 의미가 있었다.

또한 AWS RDS를 통해 DB 서버를 올리고, 해당 서버 환경에서 데이터베이스 설치와 동작 확인까지 진행했다.  
로컬이 아닌 외부 DB 환경에서 실제로 연결이 가능한 상태를 확인하면서, 이후 백엔드와 연동할 수 있는 기본 기반을 마련했다.

---

## 2. 정리한 내용

- 코드 작성 기준과 스타일 통일 방향 재확인
- 협업 시 일관된 코드 구조의 필요성 정리
- AWS RDS 기반 DB 서버 구성
- 서버 내 DB 설치 및 실행 확인
- 외부 환경에서 DB가 정상 동작하는지 점검

---

## 3. 느낀 점

협업에서는 기능 구현만큼이나 코드 스타일과 작성 기준을 맞추는 일이 중요하다는 점을 다시 느꼈다.  
또한 DB를 직접 서버 환경에 올리고 동작을 확인하는 과정에서, 개발 환경을 넘어 실제 배포와 연결되는 흐름을 조금 더 구체적으로 이해할 수 있었다.

---

## 4. 핵심 한 줄 정리

코드 컨벤션을 다시 맞추고, AWS RDS 기반 DB 서버를 구성해 실제 동작 가능한 데이터베이스 환경을 확인했다.
