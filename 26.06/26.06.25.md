# TIL - 프로젝트 분배, 이슈 관리, 목데이터 정리와 DB 세팅

## 1. 진행 내용

팀 내 역할 분배와 Git 이슈 기반 진행 사항을 정리했다.  
작업 단위를 나누고 각자 맡을 범위를 조정하면서, 이후 개발 흐름이 끊기지 않도록 정리하는 데 집중했다.

또한 목데이터에서 필요한 항목을 추려 실제 DB에 넣기 위한 형태로 가공하는 작업을 진행했다.  
단순히 데이터를 보는 수준이 아니라, 어떤 값이 실제 테이블 구조에 맞게 들어가야 하는지 기준을 잡는 과정이 중요했다.

마지막으로 전체 DB 세팅까지 진행했다.  
테이블 구조와 초기 데이터 반영까지 연결되면서, 이후 기능 개발을 바로 시작할 수 있는 기반을 마련했다.

---

## 2. 정리한 내용

- 역할 분배와 Git 이슈 기반 작업 흐름 정리
- 진행 상황 공유 방식과 작업 단위 조율
- 목데이터에서 필요한 데이터 선별
- DB에 넣기 위한 데이터 형태 점검
- 전체 DB 환경 세팅 및 초기 작업 기반 마련

---

## 3. 느낀 점

기능 개발 전에 분배, 이슈 관리, 데이터 준비, DB 세팅이 먼저 정리되어야 이후 작업 속도가 안정적으로 나온다는 점을 다시 느꼈다.  
특히 목데이터를 실제 DB 구조에 맞게 옮기는 과정은 단순 입력 작업이 아니라, 데이터 구조를 이해하는 과정에 더 가깝다고 느꼈다.

---

## 4. 핵심 한 줄 정리

역할 분배와 이슈 관리로 작업 흐름을 정리하고, 목데이터 기반 DB 초기 세팅까지 마무리해 개발 준비 단계를 구체화했다.
